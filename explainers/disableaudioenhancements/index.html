Intune Remediation

Disable Audio
Enhancements

A scriptable, enterprise-wide fix for the Zoom Rooms wireless screen share failure caused by Windows audio signal enhancements on affected Lenovo devices.

Affected Models Lenovo X1 2-in-1 Gen 9  ·  T14 Gen 5
Deployment Microsoft Intune Remediations
Requires Windows 10 / 11  ·  SYSTEM context
Background

Why this happens

Root Cause

Windows Audio Signal Enhancements (SysFx) conflict with Zoom's wireless screen share audio capture path, causing the one-click Share Screen button to fail silently.

No Native Policy

Microsoft confirmed there is no GPO or Intune CSP for this setting. It is per-endpoint, OEM-implemented, and only exposed via the registry and MMDevAPI.

Dynamic GUIDs

Each audio endpoint is assigned a unique GUID per device — even for the same model. There is no static list. The script enumerates all endpoints at runtime.

SYSTEM Required

The MMDevices registry keys have restrictive ACLs. The remediation script handles permission elevation automatically using token privileges.

Technical Details

What the scripts do

Both scripts target the same registry property on every audio endpoint (Render and Capture) found on the device. The GUID portion of the path is discovered dynamically at runtime — nothing is hardcoded.

Registry target  ·  HKLM 
SOFTWARE\Microsoft\Windows\CurrentVersion\MMDevices\Audio\
  Render\{GUID}\Properties
  Capture\{GUID}\Properties

  ; Property name (PKEY_AudioEndpoint_Disable_SysFx)
  {1da5d803-d492-4edd-8c23-e0c0ffee7f0e},5

  ; Type: REG_BINARY  (16-byte PROPVARIANT, VT_UI4)
  ; Value when disabled:
  13 00 00 00 00 00 00 00  01 00 00 00 00 00 00 00
Detection script — Checks every endpoint. Exits 0 (compliant) if all endpoints have the property set correctly; exits 1 (non-compliant) if any are missing or incorrect.
Remediation script — Writes the value to every endpoint. If a direct write fails due to ACL restrictions, it enables SeRestorePrivilege, SeBackupPrivilege, and SeTakeOwnershipPrivilege, takes ownership, grants SYSTEM Full Control, and retries. After all endpoints are updated, it restarts the Windows Audio services so changes take effect immediately. Logs to C:\ProgramData\Lenovo\AudioEnhancements\Remediation.log.
Deployment Guide

Deploy via Intune Remediations

Intune Remediations are the recommended delivery method. They run the detection script on a schedule and only invoke the remediation script when a device is found to be non-compliant — ensuring persistent enforcement even after an OEM driver update re-enables enhancements.

1

Open Intune admin center

Navigate to Devices › Scripts and remediations › Remediations, then click + Create.

2

Basics tab

  • Name: Disable Audio Enhancements (Zoom Fix)
  • Description: Disables Windows SysFx audio enhancements to fix Zoom screen share on Lenovo X1 Gen 9 / T14 Gen 5.
3

Settings tab — upload both scripts

SettingValue

Remediation script file Remediate-AudioEnhancements.ps1
Run using logged-on credentials No — runs as SYSTEM
Enforce script signature check No
Run script in 64-bit PowerShell Yes
Important: "Run using logged-on credentials" must be set to No. The remediation requires the SYSTEM token to handle registry permission elevation.
4

Assignments tab

Assign to the device group containing the affected Lenovo models. Scope tags can be applied as needed for your environment.

5

Schedule

Set the schedule to run every 24 hours (or more frequently if desired). Daily enforcement is sufficient to catch driver updates that re-enable enhancements. Click Create.

Troubleshooting

Common issues

On-device logs are written to C:\ProgramData\Lenovo\AudioEnhancements\Remediation.log and are appended on each run. Intune also surfaces exit codes and stdout in the device status view under the Remediation policy.

Symptom Likely Cause Resolution
Script exits 1 with "Cannot open Properties key for writing" A third-party security product is locking the registry key beyond what token privilege elevation can override. Re-run once; if persistent, check EDR/EPP exclusions for the MMDevices registry path.
Enhancements re-enable after a driver update OEM audio driver reinstall resets the property value. Expected behavior. The daily Remediation schedule will detect and re-apply the fix automatically.
No endpoints found / compliant immediately Device has no audio hardware or the audio driver is not installed. No action required. The detection script returns compliant when no endpoints exist.
Zoom screen share still fails after remediation Audio service restart was blocked, or Zoom requires a full sign-out/sign-in to re-initialize its audio pipeline. Have the user sign out and back in to Zoom, or restart the device. Verify the log shows all endpoints were updated successfully.
