Lenovo.Client.Update

Core Cmdlets

Get-LnvUpdate
Discover Updates
Query applicable updates for this device or a target model. Returns only needed updates by default.
Save-LnvUpdate
Download Updates
Download packages to a local directory with optional progress display. Supports custom repositories.
Install-LnvUpdate
Install Updates
Install packages silently. Enforce -VerifySignature or export install history to WMI with -ExportToWMI.
Get-LnvUpdatesRepo
Build Repository
Create a local update repository filtered by package type and reboot behavior — ideal for OSD staging.
Get-LnvUpdateSummary
Compliance Check
Get an instant snapshot of update compliance status for the current device. Useful in Intune Remediations.
Get-LnvUpdateHistory
Audit Trail
Retrieve full installation history. Filter by date range for compliance reporting and change tracking.

Lenovo.Client.Update

Complete Workflow in 3 Lines

PS update-workflow.ps1
# 1. Discover applicable updates for this machine $updates = Get-LnvUpdate $updates | Format-Table -Property Title, Category, ReleaseDate # 2. Download to a local staging directory $updates | Save-LnvUpdate -Path "C:\Lenovo\Updates" -ShowProgress # 3. Install with verbose output $updates | Install-LnvUpdate -Verbose

Tip: Filter for silent-only packages first — Where-Object { $_.Installer.Unattended } — before Save and Install.

Lenovo.Client.Update

Built for OS Deployment

PS osd-task-sequence.ps1
# Stage a local repo — exclude auto-restart packages (type 1) $repo = 'C:\Lenovo_Updates' Get-LnvUpdatesRepo -RepositoryPath $repo -RebootType '0,3,5' # First pass: install applicable updates, log history to WMI Get-LnvUpdate -Repository $repo | Install-LnvUpdate -Path $repo -ExportToWMI -Verbose # Second pass: catches chained driver dependencies Get-LnvUpdate -Repository $repo | Install-LnvUpdate -Path $repo -ExportToWMI

Run Get-LnvUpdate twice — some drivers only become applicable after a prerequisite is installed.

Lenovo.Client.Update

Enterprise-Grade Security

  • ▸▸ Digital signature verification — Every package can be tested with Test-LnvSignature before installation to confirm Lenovo authenticity.
  • ▸▸ Enforced installation signing — Pass -VerifySignature to Install-LnvUpdate to block any unsigned or tampered package from running.
  • ▸▸ Dedicated certificate DLL — Lenovo.CertificateValidation.dll provides enhanced cryptographic verification beyond what LSUClient offered.
  • ▸▸ Drop-in LSUClient migration — LCU is API-compatible. Rename cmdlets (Get-LSUpdate → Get-LnvUpdate) and immediately gain all new security features.
