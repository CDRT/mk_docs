4

BITS Transfer + Verify

Async BITS downloads with progress. SHA256 verified on completion. Bad files auto-deleted.

# Search and download in one pipeline
Search-LnvRMUpdate -MachineType '21NT' -OS 'Windows 11' |
  Where-Object Severity -eq 'Critical' |
  Save-LnvRMUpdate -Status 'Active'
Repository

Two modes. Full control.

Full Repository

Downloads installer + descriptor + readme. Self-contained, works offline. Default mode.

Hybrid Repository

Metadata only. Endpoints download installers from Lenovo at deploy time. Saves storage.

  • Multiple repositories -- switch between production, lab, staging; separate models of diferent vintage for easier retirement
  • Import existing repos -- point at an Update Retriever folder; machine types auto-discovered
  • Test / Active workflow -- stage updates, validate, then promote
  • Recycle bin delete -- removed packages go to Recycle Bin, not permanent delete
  • Network share safe -- file-level locking prevents concurrent write corruption
Intelligence

Superseded updates, flagged automatically.

  • Packages grouped by PackageName
  • Older versions identified by release date comparison
  • Superseded rows shown faded and italic in the UI
  • "Superseded By" column shows the replacement package ID
  • No auto-deletion -- admin reviews and decides
# Find superseded updates
Get-LnvRMRepoContent |
  Where-Object IsSuperseded |
  Format-Table PackageID, Version, SupersededBy

# Clean up old versions
Get-LnvRMRepoContent |
  Where-Object IsSuperseded |
  ForEach-Object {
    Remove-LnvRMUpdate -PackageID $_.PackageID
  }
Compliance

Every change. Every user. Logged.

  • Daily log files in each repository's \logs\ folder
  • Tracks downloads, deletions, status changes, and repo config
  • Username and timestamp on every entry
  • Plain text format -- grep-friendly, easy to ship to SIEM
  • Query via Get-LnvRMAuditLog cmdlet
