Hardware Driver Packs

Most OS deployment processes target model-specific hardware drivers to the device being deployed. Driver Packs simplify this by consolidating all necessary drivers into a single self-extracting archive for the specific Windows build you need.

  • ▶One file per model per Windows version
  • ▶Self-extracting — no extra tools required
  • ▶Maintained under Enterprise Management category
  • ▶Consistent with WinPE and OSD task sequences

Where to find them

Drivers & Software page on the Lenovo Support site, under the Enterprise Management category for each specific model.

https://pcsupport.lenovo.com

Hardware Support App Packs

Introduced with the "modern drivers" architecture in Windows 10 1809, Hardware Support Apps (HSAs) provide the software layer required by certain hardware components. HSA Packs bundle all model-specific apps for environments where the Microsoft Store is restricted.

  • ▶Drivers that need additional software use HSAs for control & configuration
  • ▶Normally delivered through the Microsoft Store automatically
  • ▶HSA Packs provide the sideloading files for Store-restricted deployments
  • ▶Simple self-extracting archive — same format as Driver Packs
07 / 26
Deploy

HSAs — The Easy Method

Hardware Support Apps are designed to be delivered through the Microsoft Store automatically based on the hardware drivers installed.

The simple solution

Simply turning off the Store app prevents users from installing third-party apps from the Store while still allowing the OS to install and maintain HSAs automatically. No sideloading needed.

blog.lenovocdrt.com/hsa-best-practices-guide

Microsoft documentation

Configure Access To The Microsoft Store App For Windows Devices

Microsoft Learn

When to use HSA Packs instead

Use HSA Packs when your environment blocks Store access entirely and you need to sideload apps during OSD task sequences.

Lenovo Updates Catalog

The Lenovo Updates Catalog is the Third-Party Partner Catalog available for use directly in Configuration Manager or through System Center Updates Publisher (SCUP). Same content as Lenovo Patch — deployed and managed using ConfigMgr with WSUS integrated.

  • ▶Native ConfigMgr / WSUS integration
  • ▶Same Lenovo content source as Lenovo Patch
  • ▶Familiar approval workflow for WSUS administrators

Important Notes

  • ▶Some component firmware and all dock firmware cannot be delivered via WSUS
  • ▶Requires the Lenovo Updates Catalog Agent on all patched devices for proper hardware driver detection

Notice — WSUS Deprecated

Microsoft has deprecated WSUS. No new features will be developed and it will be removed from a future Windows Server release. Plan migration to a modern patching solution.

Microsoft Blog — WSUS Deprecation

Windows Update for Business

Since 2021, Microsoft has offered per-update approval for drivers and firmware via Intune. Admins see applicable updates for their managed devices and approve what they want deployed.

  • ▶Lenovo has submitted all driver and BIOS updates to Windows Update since 2018
  • ▶Some component firmware and all dock firmware updates are not available via WU
  • ▶Limited update detail in WU can make individual approval decisions difficult

Gradual Rollout & Timeline

New updates are offered to a small subset first. Telemetry is monitored; rollout expands if all is well. It may take 30–45 days before all devices are offered the update.

Think BIOS Config Tool V2Released Nov 2025

A complete rewrite of the original HTA-based Think BIOS Config Tool, now a native PowerShell WPF GUI (ThinkBiosConfigUI.ps1) backed by the Lenovo.Bios.Config module.

  • ▶View, edit, and export BIOS settings interactively
  • ▶Import/export .ini files with encrypted Supervisor password
  • ▶Create and import password-change files
  • ▶Clear or change Supervisor password and fingerprint data
  • ▶Save and restore Custom Defaults

Intune Integration

Create Intune-friendly Win32 packages and Remediations artifacts, with optional upload via Microsoft Graph — directly from the GUI.

Installation

Install-Script 'ThinkBiosConfigUI' Install-Module 'Lenovo.Bios.Config'

Note: ThinkCentre desktop products currently not supported due to WMI BIOS Interface differences. HTA-based INI files need to be recreated.

Lenovo.Client.UpdateNew

The official Lenovo fork of the community LSUClient module — 12 public cmdlets covering the full update lifecycle with added security.

  • ▶Digital signature verification & enforcement
  • ▶-ExportToWMI writes history to root\Lenovo\Lenovo_Updates
  • ▶Full OSD Task Sequence support — create repositories dynamically

Installation

Install-Module -Name Lenovo.Client.Update Import-Module Lenovo.Client.Update -Force
# Get applicable updates for this device
$updates = Get-LnvUpdate $updates | Format-Table Title, Category
# Download to local folder
$updates | Save-LnvUpdate -Path "C:\Updates" -ShowProgress
# Install with signature verification
$updates | Install-LnvUpdate -VerifySignature -Verbose
# OSD Task Sequence — build local repository
Get-LnvUpdatesRepo -RepositoryPath 'C:\Repo' -RebootType '0,3,5'
Get-LnvUpdate -Repository 'C:\Repo' | Install-LnvUpdate -ExportToWMI -Verbose
# Silent/unattended only
Get-LnvUpdate | Where-Object { $_.Installer.Unattended } | Save-LnvUpdate | Install-LnvUpdate
